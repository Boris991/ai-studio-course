## MCP-серверы

В этой директории находятся несколько MCP-серверов, которые используются в примерах курса. Они основаны на библиотеке `FastMCP`.

### Основные серверы

Каждый сервер представляет собой независимый сервис с определенным набором инструментов и может быть запущен отдельно. Для запуска серверов рекомендуется использовать отдельную виртуальную машину в Yandex Cloud.

-   `arxiv_mcp.py`: MCP-сервер для поиска и получения информации о научных статьях на **arXiv**.
    -   **Инструменты**: `search_arxiv`, `get_paper_details`, `search_by_author`, `search_recent`.
    -   **Запуск**: `python arxiv_mcp.py` (по умолчанию на порту 8000).

-   `medrxiv_mcp.py`: MCP-сервер для поиска препринтов на **medRxiv**, архиве медицинских статей.
    -   **Инструменты**: `search_medrxiv`.
    -   **Запуск**: `python medrxiv_mcp.py` (по умолчанию на порту 8001).

-   `notes.py`: MCP-сервер для ведения личных заметок.
    -   **Инструменты**: `add_note`, `list_notes`, `search_notes` и др.
    -   **Запуск**: `python notes.py` (по умолчанию на порту 8000, может конфликтовать с `arxiv_mcp.py` при одновременном запуске вручную).

Для запуска любого из этих серверов на виртуальной машине, необходимо:
* убедиться, что установлено окружение для работы с Python, например, miniconda
* клонировать этот репозиторий
* перейти в директорию `5-mcp/servers` и выполнить соответствующую команду.

### Шлюз `mcp_gateway.py`

`mcp_gateway.py` — это скрипт, который позволяет запускать все сервера одной командой python. Он автоматически обнаруживает все MCP-серверы в директории и может запускать их в двух режимах.

#### Режим `combined` (по умолчанию)

В этом режиме шлюз создает **один объединенный сервер**, который включает в себя инструменты из всех найденных MCP-серверов. Чтобы избежать конфликтов имен, названия инструментов автоматически получают префиксы, например: `arxivresearch__search_arxiv` или `personalnotes__add_note`.

**Запуск:**
```bash
# Запустит один сервер на порту 8000 со всеми инструментами
python mcp_gateway.py --mode combined
```
Или просто:
```bash
python mcp_gateway.py
```

#### Режим `multi`

Этот режим позволяет **запустить все серверы одновременно одной командой**. Шлюз запускает каждый обнаруженный MCP-сервер в отдельном потоке на своем порту. Порты назначаются последовательно, начиная с базового (по умолчанию 8000).

-   `arxiv_mcp.py` -> `http://...:8000`
-   `medrxiv_mcp.py` -> `http://...:8001`
-   `notes.py` -> `http://...:8002`
-   и т.д.

Это самый удобный способ запустить всю инфраструктуру для демонстрации работы с несколькими агентами.

**Запуск:**
```bash
# Запустит каждый сервер на своем порту
python mcp_gateway.py --mode multi
```
